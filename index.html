Resumo

Em toda minha vida de carreira profissional na área de tecnologia, a dor do cliente é a principal responsabilidade para o consultor entender e resolver.

Enquanto trabalhava na Mix Tecnologia, ministrei diversos treinamentos dos sistemas disponibilizados na época. Sempre ouvi e compreendi todas as sugestões para melhorar o sistema.

Depois da Mix Tecnologia, trabalhei como consultor de BI, e era uma boa prática para cada projeto ter apenas um desenvolvedor. Portanto, tive que entrar em contato com o cliente desde o levantamento de requisitos, desenvolvendo artefatos como: Data Warehouse, cubos OLAP e dashboards. Trabalhei em projetos de BI para empresa de televisão, plano de saúde e para prefeituras da cidade. Todos os projetos de empresas que trabalhei na Fábrica de Negócio foram desenvolvidos com ferramentas da Suite Pentaho.

Na Accenture, fui analista sênior em um projeto de Big Data, trazendo para um Data Lake Hadoop alguns dados de Mainframe, arquivos de texto (CSV ou TXT) e outros bancos de dados como SQL Server e Oracle. Para que esse processo aconteça com dados de Mainframe e arquivos texto, é necessário alimentar um portal em C# e, a partir disso, executar alguns scripts em Shell em um servidor linux. Para cada processo é alimentada uma tabela e utilizávamos em média 18 horas. Mas em Python, Selenium, Pandas e Pyodbc minha equipe e eu reduzimos esse tempo para alguns minutos para cada tabela. Em outra situação, dados em SQL Server e Oracle minha equipe e eu desenvolvemos um Shell Script usando algumas ferramentas Hadoop (Sqoop e Hive) conectando-se em banco de dados e o retorno da consulta SQL era transferido para Data Lake.
